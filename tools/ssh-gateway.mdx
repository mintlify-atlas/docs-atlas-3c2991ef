The **rexec-ssh** gateway enables SSH access to Rexec, allowing users to connect with a simple `ssh` commandвҖ”no CLI installation required. Inspired by [terminal.shop](https://terminal.shop), it provides instant access to the Rexec TUI dashboard and terminal environments through standard SSH.

## Overview

The SSH gateway provides:

- **Zero-install access** - Just `ssh rexec.io` from any terminal
- **Universal compatibility** - Works on Linux, macOS, Windows (WSL), mobile
- **Guest sessions** - Immediate access without pre-registration
- **Account linking** - Optional authentication for persistent data
- **TUI dashboard** - Full Bubble Tea interface over SSH
- **Terminal bridging** - Direct container/agent access

<Note>
  The SSH gateway is built with [Wish](https://github.com/charmbracelet/wish), Charm's SSH server library, providing enterprise-grade security and session management.
</Note>

## Installation

<Steps>
  <Step title="Build from Source">
    Clone the repository and build the gateway:

    ```bash
    git clone https://github.com/PipeOpsHQ/rexec.git
    cd rexec
    make ssh-gateway
    ```

    This creates `bin/rexec-ssh`.
  </Step>

  <Step title="Generate Host Key">
    Create an SSH host key for the server:

    ```bash
    ssh-keygen -t ed25519 -f ~/.ssh/rexec_host_key -N ""
    ```
  </Step>

  <Step title="Start the Gateway">
    Run the SSH gateway:

    ```bash
    ./bin/rexec-ssh --port 2222 --host-key ~/.ssh/rexec_host_key
    ```
  </Step>

  <Step title="Connect">
    From any client:

    ```bash
    ssh -p 2222 localhost
    ```
  </Step>
</Steps>

### Build for All Platforms

make ssh-gateway-all

## Authentication Methods

### Zero-Config Access (Public Key)

Like terminal.shop, the gateway **accepts ANY valid SSH public key** without pre-registration:

```bash
# Just connect - no setup required!
ssh rexec.io

# Server accepts your existing SSH key automatically
# Your key fingerprint becomes your identity
```

**How it works**:

1. User runs `ssh rexec.io`
2. SSH client offers user's default public key (e.g., `~/.ssh/id_ed25519.pub`)
3. Server **accepts any valid key**
4. Server extracts fingerprint (e.g., `SHA256:abc123...`)
5. Fingerprint lookup:
   - **Known**: Load existing user session
   - **Unknown**: Create guest session

<Info>
  First-time users get a guest session immediately. Return visits with the same key resume their session.
</Info>

### Account Linking

Link your SSH key to a Rexec account for full access:

<Tabs>
  <Tab title="In TUI">
    1. Connect via SSH: `ssh rexec.io`
    2. Press **L** for "Link Account"
    3. Enter your email/token
    4. Verify via email link
    5. Reconnect - now authenticated!
  </Tab>

  <Tab title="Via CLI">
    ```bash
    # Get your SSH key fingerprint
    ssh-keygen -lf ~/.ssh/id_ed25519.pub
    
    # Link via CLI
    rexec ssh-key link --fingerprint "SHA256:abc123..."
    ```
  </Tab>

  <Tab title="Via Web UI">
    1. Go to Settings вҶ’ SSH Keys
    2. Add your fingerprint
    3. Next SSH connection will be linked
  </Tab>
</Tabs>

## Usage Examples

### First-Time User Flow

```bash
$ ssh rexec.io
# 1. SSH handshake - client offers ~/.ssh/id_ed25519.pub
# 2. Server ACCEPTS the key (any valid key works)
# 3. Server extracts fingerprint: SHA256:xYz789...
# 4. Fingerprint lookup вҶ’ not found вҶ’ new guest session
# 5. Create ssh_fingerprints record
# 6. Initialize TUI with guest dashboard
# 7. User sees: "Welcome! Link your account for full access"
# 8. Guest can explore demo, create 1 trial terminal
```

### Returning User (Linked)

```bash
$ ssh rexec.io
# 1. SSH handshake - server accepts key
# 2. Fingerprint lookup вҶ’ found вҶ’ user_id = abc123
# 3. Load user's session, permissions, preferences
# 4. TUI shows personalized dashboard with all terminals/agents
# 5. User selects a terminal...
# 6. TUI connects to container WebSocket
# 7. PTY bridge established
# 8. User is now in container shell

# Press Ctrl+] to return to dashboard
# Press 'q' to disconnect
```

### Quick Terminal Access

```bash
# Launch SSH, press '1' to connect to first terminal
ssh rexec.io

# Or jump directly
ssh terminal-abc123@rexec.io
```

## Database Schema

The gateway uses PostgreSQL for session management:

```sql
-- SSH fingerprints - can exist without a linked user (guest sessions)
CREATE TABLE ssh_fingerprints (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    fingerprint TEXT NOT NULL UNIQUE,          -- SHA256:abc123...
    user_id UUID REFERENCES users(id),         -- NULL for unlinked guests
    public_key TEXT,                           -- Optional, for display
    key_type TEXT,                             -- ed25519, rsa, etc.
    first_seen_at TIMESTAMPTZ DEFAULT NOW(),
    last_seen_at TIMESTAMPTZ DEFAULT NOW(),
    last_ip TEXT,
    session_count INT DEFAULT 0
);

-- Guest session data (for unlinked fingerprints)
CREATE TABLE ssh_guest_sessions (
    fingerprint_id UUID REFERENCES ssh_fingerprints(id),
    preferences JSONB DEFAULT '{}',
    demo_terminals_created INT DEFAULT 0,
    PRIMARY KEY (fingerprint_id)
);
```

## API Endpoints

The gateway communicates with the Rexec API:

### SSH Key Management

```
POST   /api/ssh-keys           # Register new SSH key
GET    /api/ssh-keys           # List user's SSH keys
DELETE /api/ssh-keys/:id       # Remove SSH key
```

### Internal SSH Endpoints

```
POST   /api/internal/ssh/auth        # Validate SSH key/token
POST   /api/internal/ssh/session     # Create session for SSH user
GET    /api/internal/ssh/terminals   # List terminals for SSH session
```

## Deployment

### Docker Compose

Add to your `docker-compose.yml`:

```yaml
rexec-ssh:
  build:
    context: .
    dockerfile: Dockerfile.ssh
  ports:
    - "2222:2222"
  volumes:
    - ./ssh_host_key:/etc/rexec/ssh_host_key:ro
  environment:
    - SSH_GATEWAY_API_URL=http://rexec-api:8080
    - SSH_GATEWAY_MAX_CONNECTIONS=1000
  depends_on:
    - rexec-api
```

### Systemd Service

Create `/etc/systemd/system/rexec-ssh.service`:

```ini
[Unit]
Description=Rexec SSH Gateway
After=network.target

[Service]
Type=simple
User=rexec
WorkingDirectory=/opt/rexec
EnvironmentFile=/etc/rexec/gateway.env
ExecStart=/usr/local/bin/rexec-ssh
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

Start the service:

```bash
sudo systemctl daemon-reload
sudo systemctl enable rexec-ssh
sudo systemctl start rexec-ssh
sudo systemctl status rexec-ssh
```

## Security

### Threat Mitigation

| Concern | Mitigation |
|---------|------------|
| **Brute force** | Rate limiting, fail2ban integration, progressive delays |
| **Anonymous abuse** | Guest sessions limited (1 demo terminal, no agents) |
| **Key theft** | Keys fingerprinted; users can revoke; last-used tracking |
| **Session hijacking** | Sessions tied to SSH connection; auto-expire on disconnect |
| **Resource exhaustion** | Connection limits, idle timeouts, memory caps per session |
| **Audit trail** | All connections logged with IP, fingerprint, actions |

### Rate Limiting

Set connection limits per IP:

```bash
export SSH_GATEWAY_RATE_LIMIT=10      # connections per minute per IP
export SSH_GATEWAY_BURST=5
```

### Session Timeouts

Auto-disconnect idle sessions:

```bash
export SSH_GATEWAY_SESSION_TIMEOUT=24h  # Max session duration
export SSH_GATEWAY_IDLE_TIMEOUT=30m     # Idle timeout
```

### Audit Logging

All connections are logged with:
- IP address
- SSH key fingerprint
- User actions
- Terminal access
- Timestamps

## Troubleshooting

### Connection Refused

**Check if gateway is running**:

```bash
sudo systemctl status rexec-ssh
ss -tuln | grep 2222
```

### Key Not Accepted

**Verify SSH key exists**:

```bash
ls -la ~/.ssh/id_*.pub
```

**Generate new key if needed**:

```bash
ssh-keygen -t ed25519 -C "your@email.com"
```

### Permission Denied

**Check host key file**:

```bash
ls -la ~/.ssh/rexec_host_key
chmod 600 ~/.ssh/rexec_host_key
```

### TUI Not Loading

**Verify API connectivity**:

```bash
curl http://localhost:8080/api/profile \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### Port Already in Use

**Find conflicting process**:

```bash
sudo lsof -i :2222
```

**Use different port**:

```bash
rexec-ssh --port 2223
```

The **rexec-tui** is a beautiful, interactive terminal user interface for managing your Rexec environments. Built with [Bubble Tea](https://github.com/charmbracelet/bubbletea), it provides a native-feeling terminal experience with real-time updates and keyboard-driven navigation.

## Overview

The TUI provides a full-featured dashboard directly in your terminal, allowing you to:

- View and manage all terminals and agents
- Browse and run code snippets
- Create new terminals
- Connect to running containers
- Handle MFA-protected terminals
- Monitor terminal status in real-time

<Note>
  The TUI is perfect for users who prefer keyboard-driven workflows and want instant access to their Rexec environment without opening a browser.
</Note>

## Installation

<Steps>
  <Step title="Build from Source">
    Clone the Rexec repository and build the TUI:

    ```bash
    git clone https://github.com/PipeOpsHQ/rexec.git
    cd rexec
    make tui
    ```

    The binary will be created at `bin/rexec-tui`.
  </Step>

  <Step title="Install to PATH">
    Move the binary to your system path:

    ```bash
    sudo mv bin/rexec-tui /usr/local/bin/
    chmod +x /usr/local/bin/rexec-tui
    ```
  </Step>

  <Step title="Launch the TUI">
    Run the TUI from anywhere:

    ```bash
    rexec-tui
    ```
  </Step>
</Steps>

### Build for All Platforms

To build cross-platform binaries:

```bash
make tui-all-platforms
```

This creates binaries in `downloads/` for:
- `rexec-tui-linux-amd64`
- `rexec-tui-linux-arm64`
- `rexec-tui-darwin-amd64`
- `rexec-tui-darwin-arm64`

## Configuration

The TUI uses the same configuration as the CLI, stored at `~/.rexec/config.json`.

### Configuration File

```json
{
  "host": "https://rexec.pipeops.io",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "username": "alice",
  "email": "alice@example.com",
  "tier": "pro"
}
```

### Environment Variables

Override configuration with environment variables:

| Variable | Description | Example |
|----------|-------------|----------|
| `REXEC_HOST` | API server URL | `https://rexec.pipeops.io` |
| `REXEC_TOKEN` | Authentication token | `eyJhbGci...` |

<CodeGroup>
```bash Environment Override
# Set custom host
export REXEC_HOST="https://rexec.company.com"
export REXEC_TOKEN="your-jwt-token"

rexec-tui
```

```bash One-time Override
REXEC_HOST="https://staging.rexec.io" rexec-tui
```
</CodeGroup>

## Authentication

Before using the TUI, authenticate with the CLI:

```bash
# Interactive browser login
rexec login

# Or with token
rexec login --token eyJhbGci...
```

The TUI will read credentials from `~/.rexec/config.json`.

<Warning>
  If you're not logged in, the TUI will display an error: **"Not logged in. Run 'rexec login' first."**
</Warning>

## Features

### Real-time Status

The TUI shows live status with color indicators:

- **● Green** - Running
- **○ Yellow** - Stopped
- **○ Red** - Error

### Quick Connect

Connect to terminals by number directly from the dashboard:

```bash
# Press '1' to connect to first terminal
# Press '2' to connect to second terminal
# etc.
```

This launches `rexec connect <container-id>` in a new shell.

### Terminal Creation

Press **c** to create a new terminal with default settings:

- **Name**: `terminal-<timestamp>`
- **Image**: `ubuntu`
- **Role**: `default`

<Note>
  For advanced options (custom images, memory, CPU), use the CLI: `rexec create --name mydev --image alpine --memory 1gb --cpu 1`
</Note>

### MFA Support

When a terminal has MFA enabled (indicated by 🔒):

1. Select the terminal
2. Enter your 6-digit authenticator code
3. Press **Enter** to verify
4. On success, connects to the terminal

### Agent Terminals

Agent-connected machines appear in the list with `agent:` prefix:

```
● 5. my-laptop          agent      [agent:abc123]
```

Connect to agents the same way as containers.

## API Integration

The TUI communicates with the Rexec API:

### Endpoints Used

| Endpoint | Purpose |
|----------|----------|
| `GET /api/containers` | Fetch terminals and agents |
| `GET /api/snippets` | Fetch user snippets |
| `POST /api/containers` | Create new terminal |
| `POST /api/security/terminal/:id/mfa-verify` | Verify MFA code |

### Response Handling

The TUI handles both wrapped and legacy API responses:

```go
// Wrapped response (current API)
{
  "containers": [
    { "id": "abc", "name": "dev", ... }
  ]
}

// Legacy response (older versions)
[
  { "id": "abc", "name": "dev", ... }
]
```

## Troubleshooting

### "Not logged in" Error

**Solution**: Authenticate first:

```bash
rexec login
```

### TUI Not Found

**Solution**: Add to PATH or use full path:

```bash
# Option 1: Add to PATH
sudo mv bin/rexec-tui /usr/local/bin/

# Option 2: Use full path
./bin/rexec-tui
```

### Connection Errors

**Check host configuration**:

```bash
cat ~/.rexec/config.json
```

Ensure `host` points to the correct API server.

### Display Issues

Ensure your terminal supports:
- True color (24-bit)
- UTF-8 encoding
- Minimum 80x24 size

**Test terminal capabilities**:

```bash
echo $TERM
# Should show: xterm-256color, screen-256color, or similar
```

### Slow Performance

**Reduce API calls** by avoiding frequent refreshes. The TUI auto-loads data on startup.

## Advanced Usage

### Custom API Host

Point the TUI to a self-hosted Rexec instance:

```bash
export REXEC_HOST="https://rexec.company.internal"

```

### Running in Screen/Tmux

The TUI works seamlessly in multiplexers:

```bash
# In tmux
tmux new-session -s rexec
rexec-tui

# In screen
screen -S rexec
rexec-tui
```

### SSH Over Jump Host

Access the TUI on a remote server:

```bash
ssh user@jumphost
rexec-tui
```

All three commands attempt to find and launch `rexec-tui`.

## Build Configuration

From the Makefile:

```makefile
# Build TUI for local platform
tui:
	@echo "Building $(TUI_NAME)..."
	$(GOBUILD) -ldflags "-X main.Version=$(VERSION)" -o bin/$(TUI_NAME) ./cmd/rexec-tui

# Build for all platforms
tui-all-platforms: downloads-dir
	@echo "Building TUI for all platforms..."
	@for platform in $(PLATFORMS); do \
		os=$$(echo $$platform | cut -d- -f1); \
		arch=$$(echo $$platform | cut -d- -f2); \
		echo "  Building rexec-tui-$$platform..."; \
		CGO_ENABLED=0 GOOS=$$os GOARCH=$$arch $(GOBUILD) -ldflags "-X main.Version=$(VERSION) -s -w" \
			-o $(DOWNLOADS_DIR)/rexec-tui-$$platform ./cmd/rexec-tui; \
	done
```

## Comparison: TUI vs Web UI

| Feature | TUI | Web UI |
|---------|-----|--------|
| **Launch Speed** | Instant | Requires browser |
| **Resource Usage** | Minimal | Moderate |
| **Keyboard-driven** | Yes | Partial |
| **Real-time Updates** | Manual refresh | WebSocket live |
| **Terminal Access** | Via `rexec connect` | Embedded xterm.js |
| **Snippet Execution** | CLI integration | One-click |
| **MFA Support** | Yes | Yes |
| **Best For** | CLI power users | Visual workflows |

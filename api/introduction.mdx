## Overview

The Rexec API is a REST-based interface that provides programmatic access to all Rexec features. Use it to create and manage sandboxed containers, execute commands, handle files, and connect to interactive terminal sessions.

<Note>
  The API is the foundation for all official [SDKs](/sdks/overview). If you're using a supported language, we recommend using the SDK for a better developer experience.
</Note>

## Base URL

All API requests are made to:

```
https://rexec.pipeops.io/api
```

For self-hosted instances, replace the domain with your instance URL.

## API Endpoints

The Rexec API provides the following endpoint categories:

### Containers

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/containers` | List all containers |
| `POST` | `/api/containers` | Create a container |
| `GET` | `/api/containers/:id` | Get container details |
| `DELETE` | `/api/containers/:id` | Delete a container |
| `POST` | `/api/containers/:id/start` | Start a container |
| `POST` | `/api/containers/:id/stop` | Stop a container |

### Files

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/containers/:id/files/list` | List files |
| `GET` | `/api/containers/:id/files` | Download a file |
| `POST` | `/api/containers/:id/files` | Upload a file |
| `POST` | `/api/containers/:id/files/mkdir` | Create directory |
| `DELETE` | `/api/containers/:id/files` | Delete a file |

### WebSocket

| Endpoint | Description |
|----------|-------------|
| `/ws/terminal/:containerId` | Terminal connection |

### Authentication

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/auth/guest` | Guest login |
| `GET` | `/api/auth/oauth/url` | Get OAuth URL |
| `GET` | `/api/auth/callback` | OAuth callback |
| `GET` | `/api/profile` | Get user profile |

## Request Format

All `POST` and `PUT` requests should include a `Content-Type: application/json` header and a JSON body.

<CodeGroup>
```bash cURL
curl -X POST https://rexec.pipeops.io/api/containers \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "image": "ubuntu:24.04",
    "name": "my-container"
  }'
```

```javascript JavaScript
const response = await fetch('https://rexec.pipeops.io/api/containers', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    image: 'ubuntu:24.04',
    name: 'my-container'
  })
});

const container = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://rexec.pipeops.io/api/containers',
    headers={
        'Authorization': 'Bearer YOUR_TOKEN',
        'Content-Type': 'application/json'
    },
    json={
        'image': 'ubuntu:24.04',
        'name': 'my-container'
    }
)

container = response.json()
```
</CodeGroup>

## Response Format

All responses are returned as JSON. Successful responses typically return a `2xx` status code:

```json
{
  "id": "abc123-def456",
  "name": "my-container",
  "image": "ubuntu:24.04",
  "status": "running",
  "created_at": "2024-01-15T10:30:00Z"
}
```

## Error Responses

Errors return appropriate HTTP status codes with a JSON body:

```json
{
  "error": "Container not found",
  "code": 404
}
```

### Common Status Codes

| Code | Description |
|------|-------------|
| `200` | Success |
| `201` | Created |
| `400` | Bad Request - Invalid input |
| `401` | Unauthorized - Invalid or missing token |
| `403` | Forbidden - Access denied |
| `404` | Not Found |
| `423` | Locked - Session locked (screen lock enabled) |
| `429` | Too Many Requests - Rate limit exceeded |
| `500` | Internal Server Error |

<CodeGroup>
```go Go
container, err := client.Containers.Get(ctx, "invalid-id")
if err != nil {
    if apiErr, ok := err.(*rexec.APIError); ok {
        fmt.Printf("API Error %d: %s\n", apiErr.StatusCode, apiErr.Message)
    }
}
```

```javascript JavaScript
try {
  await client.containers.get('invalid-id');
} catch (error) {
  if (error instanceof RexecError) {
    console.error(`API Error ${error.statusCode}: ${error.message}`);
  }
}
```

```python Python
from rexec import RexecException

try:
    container = await client.containers.get("invalid-id")
except RexecException as e:
    if e.status_code:
        print(f"API Error {e.status_code}: {e}")
```
</CodeGroup>

## Getting Started

<Steps>
  <Step title="Get your API token">
    Generate a token from your [Rexec dashboard](https://rexec.pipeops.io/settings) or use guest login for testing.
  </Step>
  
  <Step title="Make your first request">
    Try listing your containers:
    
    ```bash
    curl https://rexec.pipeops.io/api/containers \
      -H "Authorization: Bearer YOUR_TOKEN"
    ```
  </Step>
  
  <Step title="Create a container">
    Create your first sandboxed environment:
    
    ```bash
    curl -X POST https://rexec.pipeops.io/api/containers \
      -H "Authorization: Bearer YOUR_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{"image": "ubuntu:24.04"}'
    ```
  </Step>
</Steps>

## Quick Example

Here's a complete example that creates a container and executes a command:

<CodeGroup>
```bash cURL
# Create container
CONTAINER_ID=$(curl -s -X POST https://rexec.pipeops.io/api/containers \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"image": "ubuntu:24.04"}' | jq -r '.id')

# Start container
curl -X POST https://rexec.pipeops.io/api/containers/$CONTAINER_ID/start \
  -H "Authorization: Bearer $TOKEN"

# Execute command
curl -X POST https://rexec.pipeops.io/api/containers/$CONTAINER_ID/exec \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"command": ["echo", "Hello from Rexec!"]}'

# Clean up
curl -X DELETE https://rexec.pipeops.io/api/containers/$CONTAINER_ID \
  -H "Authorization: Bearer $TOKEN"
```

```javascript JavaScript
// Create container
const container = await fetch('https://rexec.pipeops.io/api/containers', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ image: 'ubuntu:24.04' })
}).then(r => r.json());

// Start container
await fetch(`https://rexec.pipeops.io/api/containers/${container.id}/start`, {
  method: 'POST',
  headers: { 'Authorization': `Bearer ${token}` }
});

// Execute command
const result = await fetch(`https://rexec.pipeops.io/api/containers/${container.id}/exec`, {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ command: ['echo', 'Hello from Rexec!'] })
}).then(r => r.json());

console.log(result.stdout);

// Clean up
await fetch(`https://rexec.pipeops.io/api/containers/${container.id}`, {
  method: 'DELETE',
  headers: { 'Authorization': `Bearer ${token}` }
});
```

```python Python
import requests

# Create container
response = requests.post(
    'https://rexec.pipeops.io/api/containers',
    headers={'Authorization': f'Bearer {token}'},
    json={'image': 'ubuntu:24.04'}
)
container = response.json()

# Start container
requests.post(
    f'https://rexec.pipeops.io/api/containers/{container["id"]}/start',
    headers={'Authorization': f'Bearer {token}'}
)

# Execute command
result = requests.post(
    f'https://rexec.pipeops.io/api/containers/{container["id"]}/exec',
    headers={'Authorization': f'Bearer {token}'},
    json={'command': ['echo', 'Hello from Rexec!']}
).json()

print(result['stdout'])

# Clean up
requests.delete(
    f'https://rexec.pipeops.io/api/containers/{container["id"]}',
    headers={'Authorization': f'Bearer {token}'}
)
```
</CodeGroup>

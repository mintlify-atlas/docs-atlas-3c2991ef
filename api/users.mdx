## Overview

User endpoints handle authentication, profile management, and session control. Rexec supports both OAuth (via PipeOps) and guest sessions.

## Guest Login

Create a temporary guest session with a 50-hour time limit. Perfect for trying out Rexec without creating an account.

```bash
POST /api/auth/guest
```

<CodeGroup>
```bash cURL
curl -X POST https://api.rexec.io/api/auth/guest \
  -H "Content-Type: application/json" \
  -d '{
    "username": "demo-user",
    "email": "demo@example.com"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.rexec.io/api/auth/guest', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    username: 'demo-user',
    email: 'demo@example.com'
  })
});
const data = await response.json();
console.log('Token:', data.token);
```

```python Python
import requests

response = requests.post(
    'https://api.rexec.io/api/auth/guest',
    json={
        'username': 'demo-user',
        'email': 'demo@example.com'
    }
)
data = response.json()
print(f"Token: {data['token']}")
print(f"Expires in: {data['expires_in']} seconds")
```
</CodeGroup>

## Get OAuth URL

Get the PipeOps OAuth authorization URL for user login.

```bash
GET /api/auth/oauth-url
```

<ResponseExample>
```json 200 OK
{
  "url": "https://auth.pipeops.io/oauth/authorize?client_id=...&redirect_uri=...&state=...&code_challenge=...&code_challenge_method=S256"
}
```
</ResponseExample>

<Note>
  This endpoint is typically called by the frontend to initiate the OAuth flow. The user is redirected to PipeOps to authenticate, then back to your application with an authorization code.
</Note>

## OAuth Callback

Handles the OAuth callback after user authenticates with PipeOps. This endpoint is called automatically by the OAuth provider.

```bash
GET /api/auth/callback?code=...&state=...
```

<Note>
  This is an internal endpoint used by the OAuth flow. Your application should not call this directly.
</Note>

## User Tiers

|------|----------------|--------|-----|------|------------------|
| Guest | 1 | 512 MB | 0.5 | 2 GB | 50 hours |
| Free | 5 | 256 MB - 4 GB | 0.25 - 4 | 1 - 16 GB | 50 hours |
| Pro | 10 | up to 4 GB | up to 4 | up to 20 GB | Unlimited |
| Enterprise | 25 | up to 8 GB | up to 8 | up to 50 GB | Unlimited |

## Authentication

All API requests (except guest login and OAuth endpoints) require authentication via a JWT token:

```bash
Authorization: Bearer YOUR_TOKEN
```

### Token Claims

JWT tokens include the following claims:

- `user_id`: User ID
- `email`: Email address
- `username`: Username
- `tier`: User tier
- `guest`: Whether this is a guest session
- `subscription_active`: Whether subscription is active
- `exp`: Expiration timestamp
- `iat`: Issued at timestamp

### Token Expiration

- **Guest tokens**: 50 hours
- **Authenticated tokens**: 24 hours (default)
- **API tokens**: No expiration (revocable)

## API Tokens

For programmatic access, generate long-lived API tokens:

```bash
POST /api/tokens
```

<ParamField body="name" type="string" required>
  Token name for identification
</ParamField>

<ParamField body="scopes" type="array" required>
  Permissions: `["read", "write"]` or `["agent"]`
</ParamField>

<ParamField body="expires_at" type="timestamp">
  Optional expiration date
</ParamField>

<ResponseExample>
```json 201 Created
{
  "id": "token_xyz789",
  "token": "rexec_1234567890abcdefghijklmnopqrstuvwxyz",
  "name": "my-api-token",
  "scopes": ["read", "write"],
  "created_at": "2024-01-15T10:30:00Z"
}
```
</ResponseExample>

<Warning>
  API tokens are only shown once during creation. Save them securely. Lost tokens cannot be recovered.
</Warning>

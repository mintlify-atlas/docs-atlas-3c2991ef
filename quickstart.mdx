# Quick Start Guide

Get Rexec up and running on your machine in minutes using Docker Compose. This guide covers the fastest path to a working installation.

## Prerequisites

Before you begin, ensure you have:

<CardGroup cols={2}>
  <Card title="Docker Engine" icon="docker">
    Version 20.10 or later
    
    [Install Docker](https://docs.docker.com/get-docker/)
  </Card>
  <Card title="Docker Compose" icon="layer-group">
    Version 2.0 or later (included with Docker Desktop)
    
    Verify: `docker compose version`
  </Card>
</CardGroup>

<Note>
  **Minimum Requirements**: 2GB RAM, 10GB disk space, Linux/macOS/Windows with WSL2
</Note>

## Installation Methods

<Tabs>
  <Tab title="Docker Compose (Recommended)">
    
    The easiest way to run Rexec is with Docker Compose. This sets up the entire stack in one command.
    
    <Steps>
      <Step title="Clone the Repository">
        ```bash
        git clone https://github.com/PipeOpsHQ/rexec.git
        cd rexec/docker
        ```
      </Step>
      
      <Step title="Configure Environment Variables">
        Create a `.env` file or use the defaults:
        
        ```bash
        # Optional: Set custom credentials
        echo "JWT_SECRET=$(openssl rand -base64 32)" > .env
        echo "POSTGRES_PASSWORD=$(openssl rand -base64 32)" >> .env
        ```
        
        <Accordion title="Key Environment Variables">
          | Variable | Description | Default |
          |----------|-------------|--------|
          | `JWT_SECRET` | Secret for signing auth tokens | `rexec-dev-secret-change-me` |
          | `POSTGRES_PASSWORD` | PostgreSQL password | `rexec_dev_password` |
          | `PORT` | API listen port | `8080` |
          | `REDIS_URL` | Redis connection string | `redis://redis:6379` |
          | `DOCKER_HOST` | Remote Docker daemon (optional) | `unix:///var/run/docker.sock` |
        </Accordion>
      </Step>
      
      <Step title="Start the Stack">
        ```bash
        docker compose up --build
        ```
        
        <Note>
          First startup takes 2-3 minutes to download images and build the API. Subsequent starts are instant.
        </Note>
        
        You should see output like:
        ```
        ✅ Connected to PostgreSQL
        ✅ Connected to Docker (local socket)
        ✅ Loaded 0 existing containers from Docker
        ✅ Redis pub/sub enabled for horizontal scaling
        [GIN] Listening and serving HTTP on :8080
        ```
      </Step>
      
      <Step title="Access the UI">
        Open your browser to **http://localhost:8080**
        
        **Default Credentials:**
        - Username: `admin`
        - Password: `admin`
        
        <Warning>
          **Security**: Change the default credentials immediately in production! Go to Settings > Account after logging in.
        </Warning>
      </Step>
    </Steps>
    
  </Tab>
  
  <Tab title="Manual Installation (Development)">
    
    For development or custom deployments, you can run components individually.
    
    <Steps>
      <Step title="Install Dependencies">
        **Required:**
        - Go 1.22 or later
        - Node.js 18+ (for frontend)
        - PostgreSQL 14+
        - Redis 7+ (optional, for scaling)
        - Docker Engine
        
        ```bash
        # Verify versions
        go version        # Should be 1.22+
        node --version    # Should be v18+
        docker --version  # Should be 20.10+
        ```
      </Step>
      
      <Step title="Clone and Setup">
        ```bash
        git clone https://github.com/PipeOpsHQ/rexec.git
        cd rexec
        make setup
        ```
        
        This command:
        - Downloads Go dependencies
        - Pulls Docker base images (ubuntu, debian, arch)
        - Creates volume directory at `/var/lib/rexec`
        - Copies `.env.example` to `.env`
      </Step>
      
      <Step title="Configure Database">
        Create PostgreSQL database:
        ```bash
        psql -U postgres
        CREATE DATABASE rexec;
        CREATE USER rexec WITH PASSWORD 'rexec';
        GRANT ALL PRIVILEGES ON DATABASE rexec TO rexec;
        ```
        
        Update `.env`:
        ```bash
        DATABASE_URL=postgres://rexec:rexec@localhost:5432/rexec?sslmode=disable
        ```
      </Step>
      
      <Step title="Build and Run">
        **Backend:**
        ```bash
        make build
        ./bin/rexec server
        ```
        
        **Frontend (in a separate terminal):**
        ```bash
        cd frontend
        npm install
        npm run dev
        ```
        
        The API runs on `http://localhost:8080` and the frontend dev server on `http://localhost:5173`.
      </Step>
    </Steps>
    
  </Tab>
</Tabs>

## Verify Installation

<Steps>
  <Step title="Check Health Endpoint">
    ```bash
    curl http://localhost:8080/health
    ```
    
    Expected response:
    ```json
    {
      "status": "ok",
      "database": "connected",
      "docker": "connected",
      "containers_total": 0,
      "containers_running": 0
    }
    ```
  </Step>
  
  <Step title="Create Your First Terminal">
    Log in to the web UI at `http://localhost:8080` and click **"New Terminal"**.
    
    <CardGroup cols={3}>
      <Card title="Ubuntu 24.04" icon="ubuntu">
        Latest LTS with apt package manager
      </Card>
      <Card title="Debian Bookworm" icon="debian">
        Stable, lightweight Debian 12
      </Card>
      <Card title="Arch Linux" icon="arch-linux">
        Rolling release with pacman
      </Card>
    </CardGroup>
    
    Your terminal should be ready in **3-5 seconds**.
  </Step>
  
  <Step title="Test WebSocket Connection">
    The terminal uses WebSockets for real-time streaming. Verify it works:
    
    ```bash
    # In your Rexec terminal
    echo "Hello from Rexec!"
    ls -la
    ```
    
    You should see instant output with no lag.
  </Step>
</Steps>

## Configuration

Rexec is configured via environment variables. Here are the most important ones:

<AccordionGroup>
  <Accordion title="Server Configuration">
    ```bash
    # API server
    PORT=8080
    GIN_MODE=debug              # Use 'release' for production
    BASE_URL=http://localhost:8080
    REXEC_APP_URL=http://localhost:5173  # Frontend URL (dev)
    
    # WebSocket host for port forwarding
    REXEC_WS_HOST=localhost:8080
    
    # CORS: Comma-separated allowed origins
    ALLOWED_ORIGINS=http://localhost:5173,http://localhost:8080
    BLOCK_EMPTY_ORIGIN=false    # Set true to block CLI tools
    ```
  </Accordion>
  
  <Accordion title="Security & Authentication">
    ```bash
    # JWT secret (REQUIRED in production)
    JWT_SECRET=your-super-secret-jwt-key-change-this
    # Generate with: openssl rand -base64 32
    
    # Encryption key for sensitive data (16, 24, or 32 bytes)
    REXEC_ENCRYPTION_KEY=rexec-dev-key-do-not-use-in-prod
    # Generate with: openssl rand -base64 32
    ```
    
    <Warning>
      **Production**: Generate strong random secrets and store them securely. Never commit secrets to Git.
    </Warning>
  </Accordion>
  
  <Accordion title="Database & Redis">
    ```bash
    # PostgreSQL
    DATABASE_URL=postgres://rexec:rexec@localhost:5432/rexec?sslmode=disable
    
    # Redis (optional but recommended for production)
    REDIS_URL=redis://localhost:6379
    # Required for horizontal scaling and session management
    ```
  </Accordion>
  
  <Accordion title="Docker & Container Runtime">
    ```bash
    # Docker connection
    DOCKER_HOST=unix:///var/run/docker.sock
    # For remote Docker: tcp://remote-host:2376
    
    # TLS for remote Docker (optional)
    DOCKER_TLS_VERIFY=1
    DOCKER_CA_CERT=/path/to/ca.pem
    DOCKER_CLIENT_CERT=/path/to/cert.pem
    DOCKER_CLIENT_KEY=/path/to/key.pem
    
    # Container settings
    CONTAINER_NETWORK=rexec-network
    CONTAINER_IDLE_TIMEOUT=3600  # Seconds before cleanup
    VOLUME_PATH=/var/lib/rexec/volumes
    
    # Resource limits (defaults)
    DEFAULT_MEMORY_MB=512
    DEFAULT_CPU_SHARES=512
    DEFAULT_DISK_MB=1024
    ```
  </Accordion>
  
  <Accordion title="Storage & Recordings">
    ```bash
    # Local storage
    RECORDINGS_PATH=./data/recordings
    SCRIPTS_DIR=./scripts
    DOWNLOADS_DIR=./downloads
    
    # S3-compatible storage (optional)
    S3_BUCKET=your-bucket
    S3_REGION=us-east-1
    S3_ENDPOINT=minio:9000          # For MinIO
    S3_ACCESS_KEY_ID=minioadmin
    S3_SECRET_ACCESS_KEY=minioadmin
    S3_PREFIX=recordings
    S3_FORCE_PATH_STYLE=true        # Required for MinIO
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 8080 already in use">
    Change the port in your `.env` file:
    ```bash
    PORT=3000
    ```
    Then restart: `docker compose down && docker compose up`
  </Accordion>
  
  <Accordion title="Cannot connect to Docker daemon">
    Ensure Docker is running:
    ```bash
    docker ps  # Should show running containers
    ```
    
    If using rootless Docker or Podman, update `DOCKER_HOST` in `.env`:
    ```bash
    DOCKER_HOST=unix://$XDG_RUNTIME_DIR/docker.sock
    ```
  </Accordion>
  
  <Accordion title="PostgreSQL connection failed">
    Verify PostgreSQL is accessible:
    ```bash
    docker compose logs postgres
    ```
    
    Check credentials in `.env` match the database user/password.
  </Accordion>
  
  <Accordion title="Containers fail to start">
    Check Docker daemon resources:
    ```bash
    docker info | grep -i memory
    docker info | grep -i cpus
    ```
    
    Ensure sufficient disk space:
    ```bash
    df -h /var/lib/docker
    ```
  </Accordion>
</AccordionGroup>
